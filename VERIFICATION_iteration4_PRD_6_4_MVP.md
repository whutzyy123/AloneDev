# 迭代 4 — PRD 6.4 任务模块 MVP + 排序（验收清单）

> 范围：任务 CRUD、属特性、类型与 Severity 规则、同特性内 `sort_value` 排序（MVP 为上移/下移按钮，非拖拽）、特性详情侧栏任务进度（PRD 6.4.2 公式）。不含看板视图、批量删除、快捷键等后续项。

## 进度公式（PRD 6.4.2）

- [ ] **分母排除已取消**：某特性下仅有「已取消」任务时，详情「任务完成进度」分母为 0，百分比为 0%（或等价展示）。
- [ ] **分子为已完成**：存在「未开始 / 进行中 / 已完成」且无已取消时，分子为已完成数量，分母为非取消任务总数，百分比 = 已完成 / 分母 × 100（与特性详情文案一致）。
- [ ] **混合状态**：同时存在已完成与已取消时，分母不含已取消，分子仍为已完成数。

## 类型与 Severity

- [ ] **非 Bug 不存 Severity**：新建/编辑为 Feature（或 Refactor/Research）时保存后，数据库 `severity` 为空（或非 Bug 类型下编辑弹窗不强制严重度）。
- [ ] **Bug 严重度**：Bug 类型任务可选 Blocker / Major / Minor；新建 Bug 默认 Major（MVP）；列表「严重」列非 Bug 显示「—」。

## 状态机

- [ ] **非法切换被拦截**：在编辑弹窗中故意选择不允许的状态转移并保存，应提示错误且不入库（与 `TaskStatusTransitions` 一致）。
- [ ] **允许路径**：未开始 ↔ 进行中；进行中 → 已完成 / 已取消；已完成 / 已取消 → 进行中或未开始。

## 排序

- [ ] **手动顺序模式**：筛选为「全部」、无搜索、排序为「手动顺序」时，详情「上移」「下移」可用；调整后刷新顺序保持。
- [ ] **筛选或搜索时**：上移/下移应禁用（避免与全量 `sort_value` 语义不一致）。
- [ ] **同特性**：换项目/特性后任务列表仅显示当前特性任务；排序仅影响当前特性内任务。

## 功能

- [ ] **新建任务**：任务模块选定项目与特性 → 新建 → 列表出现新行，`sort_value` 在末尾。
- [ ] **编辑 / 删除**：详情编辑、删除后列表与特性详情进度一致更新。
- [ ] **搜索防抖**：操作栏搜索约 500ms 后过滤名称/描述。

## 架构

- [ ] **离线**：无网络请求。
- [ ] **迁移**：已有库升级到 `PRAGMA user_version = 4`，`tasks` 表含业务列与 `idx_tasks_feature_order`。
