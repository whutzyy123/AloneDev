```bash
dotnet build src/PMTool.App/PMTool.App.csproj
```

## 进度公式（6.5.2，单边关联无除零）

| 场景 | 操作 | 预期 |
|------|------|------|
| 仅特性 | 版本只关联若干特性（不关联任务）；其中部分为「已完成」或「已上线」 | 详情「综合进度」= `(Rf+0)/2`，`Rf=已完成特性数/关联特性总数`，不出现除零或异常数值。 |
| 仅任务 | 版本只关联任务（不关联特性）；部分任务「已完成」 | 综合进度 = `(0+Rt)/2`，`Rt=完成任务数/关联任务总数`。 |
| 双线 | 同时有关联特性与任务 | 与 PRD 一致：`F+T=0` 则 0%；否则 `(Rf+Rt)/2`。 |

## 状态与编辑规则

| 场景 | 预期 |
|------|------|
| 未开始 | 可编辑、可删除、可「开始版本」、可添加关联。 |
| 进行中 | 不可改名称/时间；可标记已结束/已取消；可添加关联；不可删除（尝试则仓储/提示拦截）。 |
| 已结束/已取消 | 不可编辑、不可再改状态；关联列表只读逻辑由产品后续定；导出仍可试。 |

## 删除版本解耦

删除「未开始」版本后：特性与任务数据仍存在；`release_relations` 中对应行已清理（可在 SQLite 中 `SELECT * FROM release_relations WHERE release_id=...` 为空）。

## 导出

在详情面板点击「导出报告」：应在当前账号目录下 `Report` 文件夹生成 `.xlsx`（MiniExcel），含概要工作表与关联列表。（离线、无网络。）

## 非法时间

新建/编辑时开始时间晚于结束时间：对话框关闭前校验失败，错误信息显示「开始时间不可晚于结束时间」类文案。
