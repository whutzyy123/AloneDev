# 迭代 8 验收：灵感池（PRD 6.7 / 附录 11.2）

## 构建

- [ ] 在仓库根目录执行 `dotnet build src/PMTool.App/PMTool.App.csproj -c Debug`（或等效方案）无错误、无警告。

## 数据库 / 迁移

- [ ] 打开已有账户数据库：`PRAGMA user_version` 应为 `7`。
- [ ] `ideas` 表包含：`title`、`description`、`tech_stack`、`status`、`priority`、`linked_project_id`（可空）及审计列；`linked_project_id` 允许 `NULL`。
- [ ] `idea_documents` 表存在，且对 `(idea_id, document_id)` 在 `is_deleted = 0` 下唯一（或等价唯一约束）。
- [ ] 新建库：`ProjectsSchema` 与迁移后结构一致。

## 列表与搜索

- [ ] 「灵感池」导航进入真实列表页，操作栏绑定搜索、状态筛选、排序、主按钮「新建灵感」。
- [ ] 搜索框输入后约 **500ms** 防抖再刷新列表（标题 / 描述 / 技术栈）。
- [ ] 页面内「优先级」筛选与排序（含按状态排序）生效。

## 状态与校验

- [ ] 详情区「状态」下拉可在 **待评估 / 已立项 / 已搁置** 间任意切换，保存后 `updated_at`、`row_version` 合理更新。
- [ ] 重复标题（未删除行）：插入或更新应提示中文业务错误（与特性名称唯一类似）。
- [ ] **非「已立项」** 时无法保留关联项目：在「编辑灵感」中改状态或保存后，`linked_project_id` 应为空或保存被拒绝（与 `IdeaFieldValidator` 一致）。
- [ ] **「已立项」** 时可选择关联项目，「保存关联项目」写入成功。

## 项目删除守卫（PRD 4.x / 6.7）

- [ ] 仅当灵感为 **「已立项」** 且 `linked_project_id` 指向该项目时，删除该项目应被 `ProjectDeletionGuard` 拦截。
- [ ] 若灵感仅为「待评估/已搁置」但历史上曾填过 `linked_project_id`（迁移数据）：删除该项目 **不应** 因此被拦（除非仍为已立项且指向该项目）。

## 文档关联 M:N

- [ ] 「添加关联」仅从 **未软删** 文档列表中选择；已关联的文档不再出现在备选列表中。
- [ ] 移除关联后：文档仍在「文档」模块中存在；`idea_documents` 对应行为与实现一致（软删或硬删后列表不再显示）。

## 删除灵感

- [ ] 软删灵感后，灵感不再出现在列表；其文档关联行已清理；文档实体仍存在。
