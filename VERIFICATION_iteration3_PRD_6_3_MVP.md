# 迭代 3 — PRD 6.3 特性模块 MVP 验收清单（表格优先）

> 范围：特性 CRUD、属项目、优先级（P0–P3）、状态机（禁止跨级前进；已完成/已上线可退回进行中）、同项目下名称唯一、列表默认按 `updated_at` 降序。附件仅 DB 占位列，无文件逻辑。删除特性时若存在 `tasks.feature_id` 关联且任务未软删则被拦截（任务模块完善前可通过 SQLite 手工插入占位行验证）。

## 功能验收

- [ ] **选择项目**：特性页顶部项目下拉必选上下文；无项目时提示先建项目。
- [ ] **新增特性**：操作栏「新建特性」→ 填写名称/描述/优先级 → 确认后表格中出现行，默认状态「待规划」。
- [ ] **编辑特性**：选中行 → 详情「编辑」→ 修改字段后列表与详情一致。
- [ ] **删除特性**：详情「删除」→ 二次确认 → 软删除后行消失（无关联任务时）。
- [ ] **搜索特性**：操作栏搜索约 **500ms 防抖**后按名称/描述/备注过滤（当前实现含 `notes`）。
- [ ] **筛选特性**：筛选菜单「全部 / 待规划 / 进行中 / 已完成 / 已上线」与列表一致。
- [ ] **排序特性**：排序菜单「更新时间降序 / 名称升序 / 创建时间降序」与列表顺序一致。
- [ ] **详情快速改状态**：详情中「可流转状态」下拉仅含规则允许项 →「应用状态」后立即刷新；非法路径应在保存时被拦截（见下节）。

## 数据与状态机

- [ ] **同项目名称唯一**：同一项目下新建或编辑为已存在名称（未软删）时应提示且不入库。
- [ ] **名称校验**：与项目一致的规则（空、长度>100、非法字符 `\ / : * ? " < > |`）。
- [ ] **描述长度**：超过 2000 字应提示（本迭代与长文本字段上限一致）。
- [ ] **禁止跨级前进**：在「编辑特性」对话框中故意将状态从「待规划」改为「已完成」或「已上线」并保存，应提示错误且不更新。
- [ ] **允许相邻前进**：待规划→进行中→已完成→已上线 逐步保存应成功。
- [ ] **允许退回进行中**：从「已完成」或「已上线」改为「进行中」应成功。
- [ ] **列表默认排序**：新建若干特性后，在未改排序为「名称」等时，默认以**更新时间**降序展示（新建或编辑后最近更新的在前）。

## 关联与删除保护

- [ ] **任务占位关联**：若 `tasks` 存在 `feature_id` 指向该特性且 `is_deleted=0`，删除特性应失败并提示。

**手动插入占位任务（将 FEATURE_ID、PROJECT_ID 换成实际值）**：

```sql
INSERT INTO tasks (id, project_id, feature_id, is_deleted, created_at, updated_at, row_version)
VALUES (
  lower(hex(randomblob(4)) || '-' || hex(randomblob(2)) || '-4000-8000-' || hex(randomblob(6))),
  'PROJECT_ID',
  'FEATURE_ID',
  0,
  strftime('%Y-%m-%d %H:%M', 'now'),
  strftime('%Y-%m-%d %H:%M', 'now'),
  1
);
```

然后尝试删除该特性应失败；将任务软删或清空 `feature_id` 后再删应成功。

## UI（MVP）

- [ ] **表格列**：列表区域呈现表头与行（名称 / 优先级 / 状态 / 更新时间）列对齐。
- [ ] **空状态**：当前项目无特性且无搜索/筛选时显示「当前项目暂无特性」与新建按钮。
- [ ] **无匹配**：有搜索或筛选但结果为空时显示「无匹配特性」。
- [ ] **模块切换**：侧栏「特性」进入 `FeatureListPage`；进入未实现模块时操作栏搜索/筛选/主按钮不可用（占位）。

## 架构与约束

- [ ] **100% 离线**：验证期内无网络请求（本迭代未引入 `HttpClient` 等）。
- [ ] **数据库迁移**：已有账号库在启动后应自动升级到 `PRAGMA user_version = 3`，`features` 含业务列与索引、 `tasks` 含可选 `feature_id`。
