# 迭代 7 验收：文档模块（PRD 6.6）

## 构建

- [ ] 在仓库根目录或 `src/PMTool.App`：`dotnet build -c Debug` 无错误。

## 数据库 / 迁移

- [ ] 打开已有账户数据库：`PRAGMA user_version` 应为 `6`。
- [ ] `documents` 表包含：`project_id` / `feature_id` 可空、`name`、`content`、`content_format`、`is_code_snippet`；新建库与迁移后结构一致。

## 文档树与关联

- [ ] 可创建「全局文档」：`project_id` 与 `feature_id` 均为空。
- [ ] 可创建「项目文档」：必选项目；删除项目时若仍有「项目」关联文档应受现有删除守卫约束。
- [ ] 可创建「特性文档」：必选项目与特性；列表副标题显示项目·特性。
- [ ] 筛选（全部 / 全局 / 项目 / 特性）与搜索（名称、正文）生效。

## 编辑与自动保存

- [ ] 修改标题、代码片段开关、Markdown 正文后，点「保存」可写入 SQLite；错误时顶部 ErrorBanner 提示。
- [ ] 等待 3 分钟：有未保存修改时应自动落库（观察 `updated_at` / 日志或二次打开内容）。
- [ ] 切换另一篇文档时：若有未保存修改，应先尝试保存成功后再加载新文档；若保存失败，选中项应回退到当前文档并提示。

## 粘贴图片

- [ ] 在编辑区粘贴位图：生成 `Data/{账户}/Images/{docId}_*.png`（或类似扩展名），正文插入 `![](Images/...)` 相对路径。
- [ ] 超过 50MB 或目录无写权限：不破坏剪贴板文本操作，顶部明确错误提示。

## 导出 HTML

- [ ] 「导出 HTML」：保存前若有未保存修改会先尝试保存；选择路径后生成 `.html`，同目录下 `{文件名}_files` 含引用图片副本，正文可离线打开。

## 删除

- [ ] 软删文档后，列表与详情更新；`Images` 下以 `docId_` 前缀的文件应被尝试删除（单机权限不足时静默跳过单文件不影响删除）。
