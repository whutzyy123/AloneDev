# 迭代 2 — PRD 6.2.8 最小可测子集验收清单（MVP）

> 范围：功能 8 项 + 数据校验 5 项 + 删除保护 1 项 + 空状态/详情/无匹配提示。不包含 6.2.8 中「关联特性/任务/版本跳转」「右键菜单」「加载动画」等后续迭代项。

## 功能验收

- [ ] **新增项目**：顶部「新建项目」→ 填写名称/描述 → 确认后列表出现新卡片。
- [ ] **编辑项目**：选中项目 → 详情「编辑」→ 修改后列表与详情一致（仅「进行中」可编辑）。
- [ ] **删除项目**：详情「删除」→ 二次确认 → 软删除后列表不再显示（无关联时）。
- [ ] **搜索项目**：操作栏搜索框输入关键词，**约 500ms 防抖**后列表按名称/描述过滤。
- [ ] **筛选项目**：筛选菜单选择「全部 / 进行中 / 已归档」，列表仅显示对应状态。
- [ ] **排序项目**：排序菜单切换「更新时间降序 / 名称升序 / 创建时间降序」，列表顺序变化。
- [ ] **归档项目**：进行中项目点「归档」→ 状态变为已归档，「编辑」禁用。
- [ ] **恢复项目**：已归档项目点「恢复」→ 状态变为进行中；若与现有「进行中」同名则提示冲突且不恢复。

## 数据校验验收

- [ ] **名称必填**：新建/编辑确认时，空名称应提示错误且不入库。
- [ ] **名称长度**：超过 100 字应提示错误。
- [ ] **特殊字符**：名称含 `\ / : * ? " < > |` 应提示错误。
- [ ] **同状态唯一**：两个「进行中」同名应提示不可用；可与「已归档」同名（不同状态）。
- [ ] **描述长度**：描述超过 500 字应提示错误。

## 关联与删除保护

- [ ] **删除保护**：当某项目下存在未软删的占位关联行（`features` / `tasks` / `releases` / `documents`「项目」/ `ideas` 指向该项目）时，删除应被拒绝并提示「有关联内容…」。

**手动制造占位关联示例（将 `PROJECT_ID` 换成实际 GUID）**：

```sql
INSERT INTO features (id, project_id, is_deleted, created_at, updated_at, row_version)
VALUES (
  lower(hex(randomblob(4)) || '-' || hex(randomblob(2)) || '-4000-8000-' || hex(randomblob(6))),
  'PROJECT_ID',
  0,
  strftime('%Y-%m-%d %H:%M', 'now'),
  strftime('%Y-%m-%d %H:%M', 'now'),
  1
);
```

然后尝试删除该项目，应失败；删除或软删该 feature 行后再删项目应成功。

## UI（MVP）

- [ ] **空状态**：无项目且无筛选/搜索时显示「暂无项目…」与新建按钮。
- [ ] **卡片展开详情**：点击列表卡片，右侧详情面板显示描述与统计行。
- [ ] **无匹配**：有搜索或筛选但结果为空时显示「无匹配项目」。
